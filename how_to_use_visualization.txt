# Инструкция по использованию скрипта visualize_annotations.py

Скрипт `visualize_annotations.py` предназначен для визуализации аннотаций объектов (людей) на изображениях в формате YOLO.
Скрипт использует аннотации из папки labels и соответствующие изображения из папки images, 
и создает визуализированные изображения с нарисованными ограничивающими рамками в папке visualized.

## Режимы работы скрипта

Скрипт поддерживает два режима работы:
1. **all** - визуализация всех аннотаций из указанной директории
2. **single** - визуализация одной конкретной аннотации

## Параметры командной строки

- `--mode` - режим работы: all или single (по умолчанию: all)
- `--annotation` - путь к файлу аннотации для режима single
- `--images-dir` - путь к директории с изображениями (по умолчанию: images)
- `--labels-dir` - путь к директории с аннотациями (по умолчанию: labels)
- `--output-dir` - путь для сохранения визуализированных изображений (по умолчанию: visualized)
- `--class-names` - имена классов, разделенные запятыми (по умолчанию: person,people)

## Примеры использования

### Визуализация всех аннотаций

```
python code\visualize_annotations.py --mode all
```

### Визуализация конкретной аннотации

```
python code\visualize_annotations.py --mode single --annotation labels\train00_V000_1141_squared.txt
```

### Использование с другими путями к директориям

```
python code\visualize_annotations.py --mode all --images-dir custom_images --labels-dir custom_labels --output-dir custom_output
```

### Указание других имен классов

```
python code\visualize_annotations.py --mode all --class-names pedestrian,group
```

## Результат работы скрипта

Скрипт создает визуализированные изображения в указанной директории (по умолчанию: visualized).
На изображениях рисуются ограничивающие рамки вокруг объектов из аннотаций с указанием класса объекта.

## Особенности работы

- Предполагается, что файлы аннотаций имеют формат YOLO (каждая строка: класс x_center y_center width height)
- Имена файлов изображений и аннотаций должны соответствовать друг другу (с учетом префикса train/set)
- Скрипт автоматически преобразует имена файлов аннотаций (train00_V000_XXX_squared.txt) в соответствующие имена изображений (set00_V000_XXX_squared.png)
- Если изображение для аннотации не найдено, выводится сообщение об ошибке
- При обработке всего датасета выводится статистика: сколько всего аннотаций обработано, сколько успешно визуализировано, сколько ошибок

## Требования

- Python 3.x
- OpenCV (cv2)
- NumPy
